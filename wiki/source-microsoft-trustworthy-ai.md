## Main content

Microsoft defines Trustworthy AI through three core pillars: making AI systems secure, safe, and private to protect developers and users at every layer of deployment (source: Microsoft Trustworthy AI_ Unlocking human potential starts with trust.md).

### 1. Security Capabilities
Microsoft's company-wide commitments under the Secure Future Initiative (SFI) are guided by three principles: secure by design, secure by default, and secure operations (source: Microsoft Trustworthy AI_ Unlocking human potential starts with trust.md).
- **Built-in Controls**: Foundational controls in Azure AI services target prompt injections and copyright violations (source: Microsoft Trustworthy AI_ Unlocking human potential starts with trust.md).
- **Studio Evaluations**: Azure AI Studio includes proactive risk evaluations to systematically assess model vulnerabilities.
- **Web Query Transparency**: Microsoft 365 Copilot provides administrators and users with transparency reports detailing how web search queries are used to enhance model responses (source: Microsoft Trustworthy AI_ Unlocking human potential starts with trust.md).

### 2. Safety Capabilities
Guided by the corporate Responsible AI principles (established 2018), safety focuses on building, testing, and monitoring AI to avoid harmful content, bias, and misuse (source: Microsoft Trustworthy AI_ Unlocking human potential starts with trust.md):
- **Hallucination Correction**: Introduces a real-time **Correction capability** within Azure AI Content Safety's Groundedness detection feature. This fixes hallucinated content before it is shown to the end-user (source: Microsoft Trustworthy AI_ Unlocking human potential starts with trust.md).
- **Embedded Content Safety**: Enables running Azure AI Content Safety locally on-device for edge applications where cloud connectivity is unavailable or intermittent (source: Microsoft Trustworthy AI_ Unlocking human potential starts with trust.md).
- **Protected Material Detection for Code**: A feature in Azure AI Content Safety that scans public source code in GitHub repositories to help developers make informed code-referencing and licensing choices (source: Microsoft Trustworthy AI_ Unlocking human potential starts with trust.md).

### 3. Privacy Capabilities
Privacy controls secure customer data throughout the inference and training cycles (source: Microsoft Trustworthy AI_ Unlocking human potential starts with trust.md):
- **Confidential Inferencing**: Available in Azure OpenAI Service (e.g., Whisper model), confidential endpoints ensure that customer data remains encrypted and private *during* model execution (source: Microsoft Trustworthy AI_ Unlocking human potential starts with trust.md).
- **Confidential VMs with NVIDIA H100 GPUs**: Hardware-level protection that keeps data encrypted directly on the GPU, preventing cloud providers or third parties from accessing the data (source: Microsoft Trustworthy AI_ Unlocking human potential starts with trust.md).
- **Azure OpenAI Data Zones**: Provides geographic scaling controls, ensuring data processing and storage remain strictly within EU or US borders (source: Microsoft Trustworthy AI_ Unlocking human potential starts with trust.md).

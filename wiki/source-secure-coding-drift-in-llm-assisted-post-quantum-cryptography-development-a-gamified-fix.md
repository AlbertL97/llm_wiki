# Secure Coding Drift in LLM-Assisted Post-Quantum Cryptography Development: A Gamified Fix

## Introduction
The adoption of Post-Quantum Cryptography (PQC) introduces substantial implementation complexities, such as the need for constant-time execution, resistance to side-channel attacks, and accurate parametrization. Concurrently, Large Language Models (LLMs) are becoming integral to software development workflows, including cryptographic engineering. While LLMs enhance productivity, they have demonstrated a propensity for generating insecure or suboptimal code, particularly in security-sensitive domains.

## Secure Coding Drift in PQC Model
This paper proposes a novel socio-technical vulnerability model named "Secure Coding Drift in PQC." This model captures the progressive degradation of secure coding practices that occurs with sustained reliance on LLM-generated code. Departing from static vulnerability analysis, this model conceptualizes security risk as a longitudinal behavioral phenomenon arising specifically from human-AI interaction.

## Proposed Solution: Gamified Framework
To counteract this drift, the authors introduce a gamified, LLM-augmented secure coding framework. This framework is designed to embed adversarial evaluation, provide behavioral feedback, and implement security scoring within development workflows. By adopting this approach, LLMs are reframed from passive coding assistants into active security co-pilots, contributing to safer PQC implementation in AI-mediated development environments.

## Key Contributions

*   **Vulnerability Model:** Introduction of the "Secure Coding Drift in PQC" model, focusing on the behavioral aspects of human-LLM interaction in secure coding.
*   **Mitigation Strategy:** Development of a gamified framework to actively manage and improve secure coding practices in LLM-assisted development.
*   **LLM Role:** Redefinition of LLMs' role as proactive security partners.

# Rethinking Penetration Testing for AI-Enabled Systems: From Resource Compromise to Behavioral Objective Violation

**Last updated:** 2026-07-16

## Main Findings

*   This paper proposes a shift in penetration testing for AI-enabled systems from traditional resource compromise to a focus on violating behavioral objectives.
*   It introduces new adversarial pathways specific to AI, such as prompt injection and data poisoning, that allow for behavioral manipulation without direct infrastructure compromise.
*   The paper defines AI-enabled penetration as the induction of AI-governed behavior that violates operational objectives under a defined threat model.
*   A comprehensive testing workflow is outlined to identify objectives, analyze influence surfaces, define failure criteria, and test for behavioral violations.

## Key Concepts

*   **AI-Enabled Systems:** Systems where learned models materially influence behavior affecting operational outcomes.
*   **AI-Enabled Penetration:** The feasible induction of AI-governed behavior that violates operational objectives.
*   **Adversarial Pathways:** New attack vectors like prompt injection, data poisoning, sensor manipulation, tool misuse, and agentic misalignment.
*   **Objective-Driven Behavioral Evaluation:** Reframing penetration testing to assess the impact of adversarial actions on system objectives through behavior.

## Proposed Workflow

1.  Identify operational objectives.
2.  Map AI-governed behavior.
3.  Analyze adversarial influence surfaces.
4.  Define behavioral failure criteria.
5.  Execute scenario-based tests.
6.  Report evidence linking adversarial action to objective violation.

## Example

The paper illustrates the concept with a running example involving an AI-enabled security operations center assistant, demonstrating how behavioral influence, rather than infrastructure compromise, can constitute penetration.

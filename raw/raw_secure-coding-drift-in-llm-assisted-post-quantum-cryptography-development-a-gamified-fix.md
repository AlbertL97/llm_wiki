# Secure Coding Drift in LLM-Assisted Post-Quantum Cryptography Development: A Gamified Fix

**Authors:** Not specified in the provided text.
**Date:** 2026-06-22 (based on URL)

## Summary

The transition to Post-Quantum Cryptography (PQC) presents significant implementation challenges, demanding strict adherence to constant-time execution, side-channel resistance, and precise parameterization. Large Language Models (LLMs) are increasingly integrated into software development, including cryptographic engineering, leading to improved productivity but also a tendency to generate insecure or suboptimal code, especially in security-critical areas.

This paper introduces "Secure Coding Drift in PQC," a novel socio-technical vulnerability model. This model describes the gradual erosion of secure coding practices resulting from prolonged reliance on LLM-generated code. It shifts focus from static vulnerabilities to viewing security risk as a longitudinal behavioral phenomenon arising from human-AI interaction.

To address this, the authors propose a gamified, LLM-augmented secure coding framework. This framework incorporates adversarial evaluation, behavioral feedback, and security scoring directly into development workflows. The approach re-envisions LLMs not as passive assistants, but as active security co-pilots, aiming to enhance the safety of PQC implementations in AI-mediated environments.

## Key Findings

*   **Problem:** LLMs, while boosting productivity in software development (including PQC), can introduce security vulnerabilities due to generating insecure or suboptimal code.
*   **Novelty:** Introduction of "Secure Coding Drift in PQC," a socio-technical vulnerability model that views security risk as a longitudinal behavioral phenomenon stemming from human-AI interaction.
*   **Solution:** A gamified, LLM-augmented secure coding framework designed to mitigate secure coding drift.
*   **Framework Components:** Adversarial evaluation, behavioral feedback, and security scoring integrated into development workflows.
*   **Role of LLMs:** Reframing LLMs as active security co-pilots rather than passive assistants.
*   **Goal:** To improve the safety of PQC implementation in AI-mediated development.
